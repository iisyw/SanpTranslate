## 1. 引言

### 1.1 编写目的

本文档定义 SnapTranslate 项目的测试策略、测试范围、测试环境、测试进度安排及资源分配，确保软件质量满足需求规格说明书的要求。

## 2. 测试范围

### 2.1 测试对象

| 测试对象         | 说明                                           |
|-----------------|------------------------------------------------|
| 截图功能         | 全局快捷键截图、框选、取消、自动贴图、剪贴板复制 |
| 贴图功能         | 剪贴板贴图、拖拽移动、双击关闭、右键菜单、多贴图并存 |
| OCR 翻译功能     | 本地 OCR 识别、大模型翻译、译文覆盖标签          |
| 配置管理         | API 配置读写、密钥安全存储、连接测试、快捷键自定义 |
| 系统托盘         | 托盘图标显示、菜单功能                          |
| 历史记录         | 自动记录、查看、复制、删除、清空                 |
| 国际化           | 中英文界面切换                                  |
| 非功能性         | 性能、安全性、跨平台兼容性                      |

### 2.2 不在测试范围内

- Tesseract OCR 引擎本身的识别精度（第三方组件）
- 大模型 API 服务端的正确性（第三方服务）
- 操作系统 WebView 渲染引擎的兼容性

---

## 3. 测试策略

### 3.1 测试层次

| 测试层次     | 目标                                       | 方法               | 占比   |
|-------------|--------------------------------------------|--------------------|--------|
| 单元测试     | 验证各模块函数级逻辑正确性                 | 自动化测试         | 30%    |
| 集成测试     | 验证模块间交互与 IPC 通信正确性            | 自动化测试         | 25%    |
| 系统测试     | 验证端到端功能完整性与业务流程             | 手工 + 自动化      | 30%    |
| 验收测试     | 验证是否满足 PRD 验收标准                  | 手工测试           | 15%    |

### 3.2 测试类型

| 测试类型     | 说明                                           | 优先级 |
|-------------|------------------------------------------------|--------|
| 功能测试     | 验证各功能需求是否正确实现                      | 高     |
| 性能测试     | 验证响应时间、内存占用等性能指标                | 高     |
| 安全测试     | 验证密钥存储、通信加密、无数据外传              | 高     |
| 兼容性测试   | 验证跨平台（Windows/macOS/Linux）表现           | 中     |
| 易用性测试   | 验证操作流程是否直观、提示是否清晰              | 中     |
| 稳定性测试   | 长时间运行、多贴图并存、频繁操作                | 中     |
| 边界测试     | 极端截图尺寸、空图像、网络异常等边界场景        | 低     |

## 7. 缺陷管理

### 7.1 缺陷等级

| 等级   | 定义                                           | 修复时限 |
|-------|------------------------------------------------|----------|
| 致命   | 系统崩溃、数据丢失、安全漏洞                    | 24 小时  |
| 严重   | 核心功能无法使用、翻译结果严重错误              | 48 小时  |
| 一般   | 非核心功能异常、UI 显示问题                     | 1 周     |
| 轻微   | 文案错误、样式微调                              | 下版本   |

### 7.2 缺陷生命周期

```
新建 -> 确认 -> 修复中 -> 已修复 -> 验证通过 -> 关闭
                 |                     |
                 +-> 重新打开 <--------+
```

## 8. 风险与缓解

| 风险                     | 影响 | 缓解措施                                   |
|-------------------------|------|-------------------------------------------|
| Wayland 下截图功能受限    | 高   | 优先在 X11 环境测试，Wayland 作为兼容性测试项 |
| Tesseract 依赖安装复杂    | 中   | 提供详细安装文档，CI 中预装依赖             |
| 大模型 API 不稳定         | 中   | 使用 Mock 服务器进行测试，减少对真实 API 依赖 |
| 多平台测试资源不足        | 中   | 优先保证 Windows 平台测试覆盖率             |
